# دليل إضافة أفكار جديدة وترتيب المشروع

## ١ — كيف نضيف فكرة جديدة للمشروع

### الخطوة ١: توثيق الفكرة
اكتب الفكرة في مكان واحد (مثلاً الملف `IDEAS.md` في الجذر أو هنا):

- **ماذا** تريد (ميزة، تحسين، إصلاح)؟
- **لماذا** (مشكلة تحلها أو فائدة للمستخدم)؟
- **أولوية:** عالية / متوسطة / منخفضة

### الخطوة ٢: تحديد مكان الكود
| نوع الفكرة | أين تضاف في المشروع |
|------------|----------------------|
| **صفحة/شاشة جديدة** في الواجهة | `dashboard/index.html` (قسم جديد أو تبويب) أو ملف جديد مثل `dashboard/صفحة.html` ثم إضافة رابط في القائمة |
| **API جديد** (مثلاً تقرير، عملية) | `server/routes/api.js` أو وحدة جديدة تحت `server/modules/` ثم ربطها في `server/index.js` |
| **تسجيل دخول / صلاحيات** | `server/routes/auth.js` و `server/auth/middleware.js` |
| **إعدادات عامة (علامة تجارية، عملة)** | `server/config/settings.js` وواجهة الإعدادات في الـ dashboard |
| **مخزون / حركات / تقارير مخزون** | `server/inventory/` و `server/routes/api.js` |
| **محاسبة (قيود، تقارير)** | `server/accounting/` و `server/routes/api.js` |
| **تكامل خارجي (واتساب، دفعات)** | `server/modules/` (مجلد جديد باسم الميزة) |

### الخطوة ٣: ترتيب التنفيذ
1. **الـ API أولاً:** حدد المسارات (GET/POST/PUT/DELETE) والبيانات المطلوبة والرد.
2. **الواجهة ثانياً:** استخدم `fetch(API + '/مسار')` مع `getAuthHeaders()` إن لزم.
3. **الربط:** تأكد أن الـ API مُسجَّل في `server/index.js` (مثلاً `app.use('/api/...', routes)`).

### الخطوة ٤: أمان واتساق
- استخدم `requireAuth` أو `requireAdmin` للطرق الحساسة.
- تحقق من المدخلات (وجود الحقول، الأرقام، النصوص).
- أرجع صيغة موحدة: `{ success: true, data: ... }` أو `{ success: false, error: 'رسالة' }`.

## ٣ — اقتراح ترتيب للمشروع (تنظيم أفضل)

### الهيكل الحالي (ملخص)
```
cc/
├── api/index.js          ← نقطة دخول Vercel (serverless)
├── server/
│   ├── index.js          ← Express، ربط المسارات والـ dashboard
│   ├── routes/           ← auth, api, superAdmin, import
│   ├── modules/         ← fractioning, whatsapp, barter, vision, ...
│   ├── accounting/       ← قيود، تقارير، سندات
│   ├── inventory/       ← مخزون، FIFO
│   ├── config/           ← store, settings, seed
│   └── auth/             ← JWT، صلاحيات
├── dashboard/
│   ├── index.html        ← لوحة التحكم الرئيسية (كبيرة)
│   ├── login.html
│   └── super-admin.html
├── docs/                 ← توثيق، تقارير فحص
├── vercel.json
└── render.yaml
```

### تحسينات مقترحة (بدون كسر التشغيل)

| التحسين | الفائدة |
|---------|---------|
| **تقسيم الـ dashboard إلى أقسام JS منفصلة** | بدل كود واحد ضخم في `index.html`، استخدم ملفات مثل `dashboard/js/invoice.js`, `dashboard/js/reports.js` وتضمّنها في الصفحة. يسهل الصيانة والبحث. |
| **ملف واحد للأفكار** | إنشاء `IDEAS.md` في الجذر لتسجيل الأفكار والأولويات في مكان واحد. |
| **توثيق الـ API في مكان واحد** | ملف مثل `docs/API.md` ي list كل المسارات والبارامترات والردود (يمكن توليده لاحقاً من الكود). |
| **مجلد `server/constants`** | نقل الثوابت (أدوار المستخدم، أنواع الحسابات، رموز الأخطاء) إلى ملفات مثل `roles.js`, `accountTypes.js` لاستخدامها من الـ routes والـ middleware. |
| **اختبارات بسيطة** | مجلد `tests/` مع اختبارات للـ API (مثلاً باستخدام `node --test` أو أداة خفيفة) لضمان أن التعديلات لا تكسر الوظائف الأساسية. |

### ترتيب تنفيذ التحسينات (اقتراح)
1. إنشاء **IDEAS.md** وتسجيل ٣–٥ أفكار أو تحسينات تريدها أولاً.
2. توثيق **المسارات الحالية** في `docs/API.md` (يدوياً أو لاحقاً أوتوماتيك).
3. إذا كبرت `dashboard/index.html` أكثر: **استخراج** أول قسم واحد (مثلاً الفواتير) إلى `dashboard/js/invoices.js` وتضمينه في الصفحة.
4. إضافة **constants** فقط عند الحاجة (مثلاً عند إضافة دور مستخدم جديد أو نوع معاملة جديد).

## ٤ — أولويات نموذجية لمشروع POS

- **أساسي:** تسجيل الدخول، المستخدمون، الفواتير، الأصناف، الوحدات، المخزون، التقارير الأساسية.
- **متوسط:** المرتجعات، النسخ الاحتياطي، الاستيراد، الإعدادات والعلامة التجارية.
- **متقدم:** البارتر، العملات المتعددة، الواتساب، التصنيع، المشتريات، المراقبة والتدقيق.

يمكنك نسخ القسم «قالب لتسجيل فكرة» في `IDEAS.md` واستخدامه لكل فكرة جديدة، ثم الرجوع إلى «كيف نضيف فكرة جديدة» لتنفيذها بترتيب واضح.
